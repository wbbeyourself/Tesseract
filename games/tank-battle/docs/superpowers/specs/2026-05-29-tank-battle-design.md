# 坦克大战网页版 - 设计文档

**日期**: 2026-05-29
**状态**: 已确认
**技术栈**: HTML5 Canvas + 原生 JavaScript（零依赖）

---

## 1. 项目概述

制作一个基于经典 FC 游戏《坦克大战》(Battle City) 的现代化网页版。保留经典玩法的核心乐趣，通过高清像素风视觉升级、丰富的道具系统、坦克升级技能树和渐进式 AI 难度，打造一个功能全面的单人闯关体验。

### 核心定位

- **风格**: 现代化升级版（高清像素风）
- **模式**: 单人关卡闯关
- **平台**: 桌面浏览器（PC）
- **部署**: 静态 HTML 文件，打开即玩，零外部依赖

---

## 2. 文件结构

```
tank-battle/
  index.html          ← 游戏主文件（内嵌 JS 和 CSS）
  data/
    maps.js           ← 关卡地图数据（经典 35 关 + 新设计 15-45 关）
```

游戏主体全部内嵌在 `index.html` 中，地图数据单独放在 `data/maps.js` 中避免主文件过大。所有精灵图形通过像素数据数组在代码中定义，不依赖外部图片资源。

---

## 3. 核心架构

### 3.1 游戏循环

```
初始化 → [输入处理 → 逻辑更新 → 碰撞检测 → 渲染] 循环（60fps）
```

### 3.2 模块划分

在同一 JS 文件中用注释分区组织：

| 模块 | 职责 |
|---|---|
| **GameEngine** | 游戏主循环、帧率控制(60fps)、场景管理 |
| **InputManager** | 键盘输入映射、按键状态缓存 |
| **EntityManager** | 管理所有游戏实体（坦克、子弹、道具、爆炸） |
| **PhysicsEngine** | 碰撞检测（矩形碰撞）、移动约束 |
| **MapRenderer** | 地图渲染、地形管理（砖墙、钢墙、水域、树林、冰面） |
| **AIController** | 敌人 AI 行为决策 |
| **AudioManager** | 音效和背景音乐管理 |
| **SaveManager** | localStorage 存档和排行榜 |
| **UIManager** | 菜单、HUD、暂停、关卡结算等界面 |
| **ParticleEngine** | 爆炸、火花等粒子特效 |

### 3.3 游戏状态机

```
主菜单 → 关卡加载 → 游戏进行中 →（关卡完成 → 下一关 / 游戏结束 → 结算画面）
                ↘ 暂停 → 继续/返回主菜单
```

状态转换规则：
- 主菜单 → 关卡加载：点击"开始游戏"或"继续游戏"
- 关卡加载 → 游戏进行中：地图和实体初始化完成
- 游戏进行中 → 暂停：按 P 或 ESC
- 游戏进行中 → 关卡完成：消灭所有敌人
- 游戏进行中 → 游戏结束：基地被摧毁 或 生命耗尽
- 关卡完成 → 下一关（关卡加载）：自动进入 或 显示升级界面后进入
- 游戏结束 → 结算画面：显示成绩统计

## 4. 地图系统

### 4.1 格子规格

- 26×26 格子网格（经典尺寸）
- 每格 16×16 像素基础尺寸
- 画面放大到 832×832（2x 缩放渲染）
- 基地位于底部中央（13-14 列，25 行）

### 4.2 地形类型

| 类型 | 编码 | 可通过 | 可摧毁 | 特殊效果 |
|---|---|---|---|---|
| 空地 | 0 | ✓ | - | - |
| 砖墙 | 1 | ✗ | ✓（子弹摧毁） | 分为 4 个子块，可部分摧毁 |
| 钢墙 | 2 | ✗ | ✗（Lv4 强力弹可摧毁） | - |
| 水域 | 3 | ✗ | - | 子弹可飞越，坦克不可通过 |
| 树林 | 4 | ✓ | - | 遮挡坦克（渲染在坦克层上方） |
| 冰面 | 5 | ✓ | - | 坦克经过时惯性滑动 |
| 基地 | 9 | ✗ | ✓（被摧毁则游戏结束） | 必须保护 |

### 4.3 地图编码

地图数据使用紧凑字符串编码：

```javascript
// 地形类型编码对应上表
// 每行 26 个字符，共 26 行，用 \n 分隔
const MAPS = [
  "00000000000000000000000000\n" +  // 第 1 行
  "00000000000000000000000000\n" +  // 第 2 行
  // ... 中间行 ...
  "00000011190011110000000000",     // 第 26 行（基地行）
];
```

### 4.4 关卡内容

- **第 1-35 关**：复刻经典 FC 版 35 张地图布局
- **第 36-80 关**：新设计的地图，包含多种类型：
  - 迷宫型（大量砖墙通道）
  - 开阔型（大片空地，考验走位）
  - 水域型（河流分割战场）
  - 要塞型（钢墙堡垒+多个入口）
  - 混合型（所有地形类型混搭）

## 5. 实体系统与战斗机制

### 5.1 玩家坦克

**4 级升级系统**（通过拾取星星道具升级）：

| 等级 | 速度 | 炮弹 | 特殊能力 |
|---|---|---|---|
| Lv1 普通 | 慢 | 单发普通弹 | 无 |
| Lv2 快速 | 中 | 单发快速弹 | 子弹速度提升 |
| Lv3 强力 | 中 | 双发快速弹 | 可同时发射 2 颗子弹 |
| Lv4 超级 | 快 | 双发强力弹 | 可摧毁钢墙 |

**生命与降级系统**：
- 初始 3 条命
- 被击中后降一级（不低于 Lv1）
- 降级到 Lv1 时再被击中则失去一条命
- 关卡之间保留剩余生命和当前等级

**操作控制**：
- 方向键：移动（上下左右 4 方向）
- 空格 / J：射击
- P：暂停
- ESC：返回/退出
- Shift：冲刺（需要技能树解锁）

### 5.2 敌人坦克类型

| 类型 | 颜色 | 血量 | 速度 | 特点 | 出现关卡 |
|---|---|---|---|---|---|
| 普通 | 灰色 | 1 | 慢 | 随机移动射击 | 关卡 1+ |
| 快速 | 白色 | 1 | 快 | 速度快 | 关卡 3+ |
| 强力 | 绿色 | 2 | 中 | 子弹快 | 关卡 5+ |
| 装甲 | 金色 | 4 | 慢 | 血量高 | 关卡 8+ |
| 精英 | 红色 | 3 | 中 | AI 更聪明 | 关卡 12+ |

**生成规则**：
- 每关生成固定数量敌人
- 敌人从地图顶部 3 个刷新点（左、中、右）依次出现
- 前期 4-6 个同屏，后期增加到 6-8 个同屏
- 闪烁的红色坦克被击杀后掉落道具

### 5.3 Boss 坦克

每 5 关（第 5、10、15、20... 关）出现 Boss：

- 体型是普通坦克的 2×2 倍（32×32 像素）
- 有血条显示在屏幕上方
- 特殊攻击模式（散射、冲刺）
- 击败 Boss 必定掉落一个高级道具（从护盾💎、导弹🚀、散弹🔫中随机选择）

### 5.4 子弹系统

- 子弹与地形碰撞：砖墙被摧毁（1 发），钢墙反弹（除非 Lv4 强力弹摧毁）
- 子弹与子弹碰撞：玩家子弹与敌人子弹互相抵消；同一方的子弹不互碰
- 子弹速度固定，不同等级坦克射速不同
- 子弹击中产生爆炸动画（粒子效果）
- 玩家同屏子弹上限：Lv1-2 为 1 颗，Lv3-4 为 2 颗

## 6. 道具系统

道具通过击杀闪烁的红色坦克后掉落，出现在随机位置，持续 10 秒后消失。

### 6.1 经典道具

| 道具 | 图标 | 效果 |
|---|---|---|
| 星星 | ⭐ | 坦克升一级（最高 Lv4） |
| 炸弹 | 💣 | 消灭当前屏幕所有敌人 |
| 铲子 | 🔧 | 基地周围砖墙变钢墙 10 秒 |
| 钢盔 | 🛡️ | 无敌 10 秒 |
| 时钟 | ⏰ | 冻结所有敌人 5 秒 |
| 坦克 | ❤️ | 额外一条命 |

### 6.2 新增道具

| 道具 | 图标 | 效果 |
|---|---|---|
| 散弹 | 🔫 | 下 3 次射击发射扇形 3 颗子弹 |
| 导弹 | 🚀 | 下 1 次射击发射穿透导弹，穿透砖墙 |
| 加速 | ⚡ | 移动速度翻倍 10 秒 |
| 护盾 | 💎 | 吸收 1 次伤害不降级 |

## 7. 坦克升级/技能树系统

### 7.1 经验值与升级

- 击杀敌人获得经验值 (XP)：普通 10、快速 15、强力 20、装甲 30、精英 40、Boss 100
- XP 跨关卡累积，升级后永久生效
- 升级所需 XP：`100 × 当前已消耗技能点总数`
- 每次升级获得 1 个技能点
- 每 50 关可以选择重置技能树（重新分配技能点，已消耗技能点全部返还）

### 7.2 技能树三大分支

#### 🔫 火力分支

| 等级 | 效果 |
|---|---|
| Lv1 | 子弹速度 +15% |
| Lv2 | 射速 +20% |
| Lv3 | 子弹穿透力+1（可穿透薄墙后的敌人） |
| Lv4 | 暴击几率 15%（双倍伤害） |
| Lv5 | 终极：散射弹（每次射击 3 颗子弹扇形发射） |

#### 🛡️ 防御分支

| 等级 | 效果 |
|---|---|
| Lv1 | 额外护甲 +1（被击中时先消耗护甲而不是降级，每关恢复） |
| Lv2 | 受伤无敌时间 +1 秒 |
| Lv3 | 基地防护强化（铲子道具效果时间翻倍） |
| Lv4 | 受到攻击 20% 几率完全闪避 |
| Lv5 | 终极：复活（每关 1 次免费复活，保留等级） |

#### ⚡ 机动分支

| 等级 | 效果 |
|---|---|
| Lv1 | 移动速度 +15% |
| Lv2 | 转向速度 +30%（方向切换更灵活） |
| Lv3 | 子弹弹射（子弹碰墙反弹 1 次） |
| Lv4 | 冲刺能力（按 Shift 短距离快速移动，5 秒 CD） |
| Lv5 | 终极：瞬移（按空格+方向瞬移 3 格，10 秒 CD） |

### 7.3 技能点规则

- 单个分支最多投入总技能点一半的点数（防止极端加点）
- 技能树界面在关卡间隙时展示，也可在暂停菜单中查看
- 分支内必须按顺序逐级解锁

## 8. 敌人 AI 与难度系统

### 8.1 AI 行为层级（随关卡递进）

**第 1-4 关 — 基础 AI：**
- 随机移动，遇墙转向
- 随机射击，不瞄准
- 不主动攻击基地
- 偶尔原地不动 1-2 秒

**第 5-9 关 — 中等 AI：**
- 倾向向基地方向移动（30% 概率选择基地方向）
- 朝玩家方向射击（50% 概率）
- 被击中后会尝试躲避（短距离变向）
- 快速坦克会绕路（遇到墙会尝试另一个方向）

**第 10-14 关 — 高级 AI：**
- 明确的战术行为：部分敌人直冲基地
- 瞄准玩家射击（80% 命中方向）
- 装甲坦克会做肉盾掩护其他敌人
- 学会避开子弹（检测来袭子弹方向并移动）
- 精英坦克会包抄玩家

**第 15+ 关 — 专家 AI：**
- 多坦克协同：一组吸引玩家注意，另一组偷袭基地
- 精确射击（计算提前量）
- 利用地形掩护（躲在砖墙后面射击）
- 紧急情况下自杀式攻击基地

### 8.2 AI 决策框架

每个敌人每 500ms 执行决策循环：

```
1. 生存检测 → 附近有子弹？闪避
2. 战术目标 → 当前目标（攻击玩家/攻击基地/巡逻）
3. 路径选择 → 向目标移动
4. 射击决策 → 是否开火
```

### 8.3 关卡难度参数

```
关卡 N 的难度参数:
  敌人总数 = 10 + floor(N × 1.5)
  同屏最大数 = min(4 + floor(N/4), 8)
  AI 等级权重 = 根据关卡范围分配各 AI 层级的概率
  刷新间隔 = max(1.5, 4 - N × 0.1) 秒
  敌人类型权重 = 关卡越高，高级类型出现概率越大
```

## 9. UI 与界面系统

### 9.1 主菜单

- 游戏标题 "坦克大战 BATTLE CITY"（高清像素字体）
- 菜单项：开始游戏 / 继续游戏（有存档时） / 排行榜 / 设置

- 继续游戏 / 技能树 / 操作说明 / 返回主菜单

- 音效开关和音量滑块

### 9.6 结算画面

- 总得分、总击杀数、存活关卡数

- 最高分排行（本地 Top 10）
- 保存记录到排行榜 / 返回主菜单
- 输入 3 字母名（经典街机风格）

## 11. 存档与排行榜

### 11.1 存档系统

使用 localStorage 保存游戏进度：

```javascript
localStorage 键:
  tankBattle_save     → { stage, lives, level, xp, skillPoints, skills, totalScore }
  tankBattle_settings → { sfxVolume, bgmVolume, keyBindings }
  tankBattle_records  → [{ name, score, stage, kills, date }]
```

- 每完成一关自动存档
- 继续游戏时恢复到最近完成的关卡（不保存关卡内进度）
- 恢复的状态包括：生命、等级、技能点、XP、总分

### 11.2 排行榜

- 本地 Top 10 排行
- 记录字段：排名、玩家名（3 字母）、总分、通关关卡数、总击杀数、日期
- 游戏结束时提示保存记录
- 主菜单可查看完整排行榜

## 12. 渲染系统

### 12.1 精灵图集

所有精灵通过像素数据数组在代码中定义，运行时绘制到 Canvas：

```
玩家坦克   → 4 方向 × 4 等级 × 2 帧(动画) = 32 个 16×16 精灵
敌人坦克   → 5 类型 × 4 方向 × 2 帧 = 40 个 16×16 精灵
Boss 坦克  → 4 方向 × 2 帧 = 8 个 32×32 精灵
子弹       → 4 方向 = 4 个 4×4 精灵
地形       → 6 类型 × 边缘变体 = ~30 个 16×16 精灵
道具       → 10 个道具 = 10 个 16×16 精灵
爆炸       → 5 帧动画 = 5 个 32×32 精灵
基地       → 完好/受损/摧毁 = 3 个 16×16 精灵
UI 元素    → 分数、图标等
```

### 12.2 渲染层次（从底到顶）

1. 地面层（空地、冰面）
2. 地形层（砖墙、钢墙、水域）
3. 阴影层（坦克和地形的阴影）
4. 子弹层
5. 坦克层（玩家和敌人）
6. 树林层（遮挡坦克）
7. 特效层（爆炸、粒子、光环）
8. UI 叠加层（HUD、提示文字）

### 12.3 视觉特效

| 效果 | 实现方式 |
|---|---|
| 坦克出生动画 | 闪烁星星效果，持续 2 秒 |
| 爆炸效果 | 粒子系统，5-8 个粒子向外扩散并渐隐 |
| 无敌光环 | 半透明圆形光环闪烁效果 |
| 道具闪烁 | 道具图标脉冲闪烁效果 |
| 子弹轨迹 | 淡化的拖尾效果 |
| 基地被摧毁 | 屏幕震动 + 红色闪烁 |
| Boss 血条 | 屏幕上方显示 Boss 血量条 |
