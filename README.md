# 地牢爬塔

暗黑奇幻风格的回合制文字爬塔游戏，提供两个版本：命令行版和网页版。

---

## 快速开始

### 网页版（推荐）

```bash
./web/play.sh
```

或者直接用浏览器打开 `web/index.html`。

### 命令行版

```bash
./play.sh
# 或
python main.py
```

> **环境要求**：Python 3.10+

---

## 游戏玩法

### 目标

爬过 **20 层**地牢，击败守卫，最终登顶成为地牢霸主。

### 职业选择

| 职业 | 血量 | 攻击 | 防御 | 特点 |
|------|------|------|------|------|
| 战士 | 150  | 25   | 15   | 血厚防高，稳扎稳打 |
| 法师 | 100  | 40   | 5    | 攻高血薄，爆发强 |
| 刺客 | 120  | 30   | 10   | 20% 暴击率，伤害×2 |

### 每层事件（随机3选1）

- **遭遇怪物**：必须战斗，胜利获金币，15% 概率掉落装备
- **宝箱**：随机获得金币 / 回血药水 / 装备
- **随机事件**：休息（+30HP）、陷阱（-20HP 且防御临时-5）、神秘商人

### 战斗系统

每回合三选一：

| 行动 | 效果 |
|------|------|
| 攻击 | 造成「你的攻击 - 怪物防御」点伤害，最低 1 |
| 防御 | 本回合受到的伤害减半 |
| 使用药水 | 回复 50 点血量 |

### 装备系统

装备分武器（加攻击）和防具（加防御）两类，各只能装备一件。

| 品质 | 武器加成 | 防具加成 |
|------|----------|----------|
| 普通（白）| +5 ~ 10  | +3 ~ 6   |
| 稀有（蓝）| +11 ~ 20 | +7 ~ 12  |
| 史诗（紫）| +21 ~ 30 | +13 ~ 20 |

### 升级机制

每击败 3 层怪物有 50% 概率升级：攻击和防御各 +5，最大血量 +20，血量回满。

## 网页版特效

- 暗黑奇幻主题，紫金霓虹配色
- 实体冲刺攻击动画，受击红闪抖动，死亡旋转消散
- 浮动伤害数字（暴击更大更亮）
- 血条平滑渐变（绿 → 黄 → 红）
- Canvas 粒子：升级金光爆炸、装备拾取紫光、暴击橙色粒子
- Web Audio API 合成音效，无需外部音频文件

## 项目结构

```
Tesseract/
├── main.py              # 命令行版入口
├── play.sh              # 命令行版启动脚本
├── game/                # 命令行版游戏逻辑
│   ├── character.py     # 角色与职业系统
│   ├── combat.py        # 回合制战斗
│   ├── dungeon.py       # 主游戏循环
│   ├── equipment.py     # 装备与背包
│   ├── events.py        # 非战斗事件
│   ├── monsters.py      # 怪物定义
│   └── utils.py         # 输入验证与工具函数
└── web/                 # 网页版
    ├── index.html       # 页面结构
    ├── style.css        # 主题样式与动画
    ├── play.sh          # 网页版启动脚本
    └── js/
        ├── audio.js     # Web Audio API 音效引擎
        ├── particles.js # Canvas 粒子特效
        ├── game.js      # 游戏逻辑（纯计算）
        ├── ui.js        # DOM 渲染与动画
        └── main.js      # 流程控制与事件绑定
```

## 扩展开发

代码中预留了扩展注释，以下位置可直接追加内容：

- **新增职业**：`game/character.py` → `CLASS_DEFINITIONS` / `web/js/game.js` → `CLASS_DATA`
- **新增怪物**：`game/monsters.py` → `MONSTER_POOL` / `web/js/game.js` → `MONSTER_TEMPLATES`
- **新增装备品质**：`game/equipment.py` → `QUALITY_TIERS` / `web/js/game.js` → `QUALITY_DATA`
- **新增随机事件**：`game/events.py` → `event_random()` 的 `events` 列表
